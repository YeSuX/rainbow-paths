# Rainbow Paths 首页设计书 🌈

> 产品思维版 · 基于项目书 v2024-12-05

---

## 📋 产品目标

### 核心用户任务

用户来到首页有**一个核心目标**：

> **找到可以合法登记伴侣关系的地区，并了解如何办理**

所有设计都围绕这个目标，其他一切都是噪音。

### 用户决策路径（5 秒）

```
看到首页 → 3秒理解价值 → 信任平台 → 点击查看地图
```

### 设计原则

1. **单一主 CTA**：进入地图是唯一主要行动
2. **3 秒价值传达**：不需要滚动就能理解核心价值
3. **零思考**：不需要思考"我该做什么"
4. **信任 > 美观**：建立可信度比视觉炫酷更重要

## 🎨 页面结构（精简到 4 个区块）

```
┌─────────────────────────────────────┐
│   彩虹装饰条（2px）                  │
├─────────────────────────────────────┤
│   导航栏                            │
├─────────────────────────────────────┤
│                                     │
│   【1】Hero 区：核心价值 + 主 CTA    │
│        （首屏，完整呈现）             │
│                                     │
├─────────────────────────────────────┤
│                                     │
│   【2】快速说明：我们提供什么        │
│        （3 个核心功能，一行说清）     │
│                                     │
├─────────────────────────────────────┤
│                                     │
│   【3】信任指标：为什么可信          │
│        （数字 + 承诺，紧凑呈现）      │
│                                     │
├─────────────────────────────────────┤
│                                     │
│   【4】次要 CTA：开始探索            │
│        （最后一次行动召唤）           │
│                                     │
├─────────────────────────────────────┤
│   Footer（极简）                     │
└─────────────────────────────────────┘
```

**删除的区块及原因**：

- ❌ **如何使用区**：地图本身就很直观，不需要教程
- ❌ **最新动态区**：首次用户不关心动态，放在独立页面
- ❌ **我们的承诺区**：合并到"信任指标"，更紧凑
- ❌ **数字故事独立区**：合并到"信任指标"

## 📐 详细区块设计

### 【1】Hero 区：3 秒传达核心价值

**目标**：用户看完就知道这是什么，该做什么。

**布局**（首屏完整呈现，无需滚动）

```
┌─────────────────────────────────────┐
│                                     │
│          [一句话描述产品]            │
│                                     │
│        主标题（大、清晰、直接）       │
│                                     │
│     [大号主 CTA：查看全球政策地图]    │
│                                     │
│        [信任背书：一行文字]          │
│                                     │
└─────────────────────────────────────┘
```

**内容设计**

1. **产品定位（小字）**

   ```
   全球婚姻平权政策地图
   ```

   - 字号：14px
   - 颜色：--text-tertiary
   - 位置：居中，标题上方
   - 目的：快速定位产品类别

2. **主标题（大字）**

   ```
   找到欢迎你们的地方
   ```

   - 字号：56px（桌面）/ 36px（移动）
   - 字重：700
   - 颜色：--text-primary
   - 对齐：居中
   - 目的：情感共鸣 + 功能暗示

3. **主 CTA（唯一焦点）**

   ```
   [查看全球政策地图 →]
   ```

   - 样式：超大按钮（h-16，px-12）
   - 背景：彩虹渐变（绿 → 蓝）
   - 字号：18px，字重 600
   - 位置：居中，距标题 48px
   - Hover：上移 2px + 阴影加深
   - 目的：唯一主要行动，必须显眼

4. **信任背书（一行）**
   ```
   35+ 国家/地区 · 完全免费 · 公益项目
   ```
   - 字号：14px
   - 颜色：--text-secondary
   - 位置：按钮下方 24px
   - 用 · 分隔，紧凑呈现
   - 目的：快速建立信任

**视觉**

- 背景：纯白
- 上下内边距：160px（桌面）/ 100px（移动）
- 装饰：无（或极微妙的几何图案）
- 原则：让 CTA 成为视觉焦点

**删除的元素**

- ❌ 副标题（冗余，标题已说清）
- ❌ 次要按钮（分散注意力）
- ❌ 多行描述（减少阅读负担）

---

### 【2】快速说明：我们提供什么

**目标**：让用户理解点进地图后能得到什么。

**布局**（一行三列，移动端单列）

```
┌──────────────┬──────────────┬──────────────┐
│ 📍 政策地图   │ 📋 办理指南   │ 🤝 本地资源   │
│  查看全球...  │  了解如何...  │  找到公益...  │
└──────────────┴──────────────┴──────────────┘
```

**设计规格**

- **区块标题**：无（不需要）
- **每个功能**：
  - Emoji：32px（视觉标识）
  - 标题：16px，Semibold（"政策地图"）
  - 描述：14px，灰色，1 行（最多 15 字）
  - 间距：紧凑，Emoji 与标题 8px，标题与描述 4px
- **容器**：
  - 无边框，无背景色（最简设计）
  - 或：极淡的灰色背景（#FAFAFA）
  - 内边距：24px

**内容**

| 功能        | 描述                     |
| ----------- | ------------------------ |
| 📍 政策地图 | 查看全球各地婚姻平权政策 |
| 📋 办理指南 | 了解如何登记和所需材料   |
| 🤝 本地资源 | 找到公益组织和法律支持   |

**间距**

- 上边距：80px
- 下边距：80px
- 列间距：48px

**原则**

- 一行说清，不展开
- 只说"是什么"，不说"为什么"（留给内页）

---

### 【3】信任指标：为什么可信

**目标**：用最少的信息建立信任。

**布局**（数字 + 承诺，二合一）

```
┌─────────────────────────────────────┐
│                                     │
│   [数字指标：横向 3-4 个]            │
│                                     │
│   ─────────────────                 │
│                                     │
│   [核心承诺：横向 4 个图标]          │
│                                     │
└─────────────────────────────────────┘
```

**数字指标**（横向紧凑呈现）

```
35+ 地区   |   10,000+ 伴侣   |   持续更新
```

- 字号：数字 32px Bold，说明 13px
- 颜色：数字用彩虹色，说明用灰色
- 间距：用竖线分隔（|），简洁
- 无卡片，无边框

**核心承诺**（图标 + 文字，横向）

```
🆓 免费   🙏 公益   🔒 隐私   📚 准确
```

- 只有图标 + 一个词
- 不需要解释（看不懂就算了，减少认知负担）
- 字号：20px（Emoji）+ 14px（文字）
- 间距：均匀分布

**容器**

- 背景：淡米白（#F7F6F3），与主区块区分
- 上下内边距：64px

**分割线**

- 数字与承诺之间：1px 细线，--border-default
- 上下边距：32px

**原则**

- 不需要详细说明，给个印象即可
- 信任来自专业的设计，而不是冗长的承诺

---

### 【4】次要 CTA：最后行动召唤

**目标**：给滚动到这里的用户最后一次机会。

**布局**（极简）

```
┌─────────────────────────────────────┐
│                                     │
│      [一句鼓励的话，可选 emoji]       │
│                                     │
│      [主 CTA 按钮：开始探索]          │
│                                     │
└─────────────────────────────────────┘
```

**内容**

- **文案**：

  ```
  愿每一对相爱的人，都能找到属于自己的路 🌈
  ```

  - 字号：20px
  - 字重：600
  - 颜色：--text-primary
  - 居中

- **按钮**：
  ```
  [开始探索 →]
  ```
  - 与 Hero 区相同样式
  - 尺寸可以稍小（h-12）

**间距**

- 上下内边距：80px
- 文案与按钮：32px

**原则**

- 非常简短，不拖泥带水
- 如果用户滚到这里还没点击，再给一次机会

---

### 【5】导航栏

**布局**（简洁）

```
[Logo + Rainbow Paths]          [关于] [GitHub]
```

**设计**

- 高度：64px
- 背景：透明 → 滚动后白色 + 阴影
- 只保留 2 个链接：关于、GitHub
- 删除"地图"链接（首页就是去地图的）

---

### 【6】Footer

**布局**（极简，一行或两行）

```
© 2024 Rainbow Paths · 公益项目
关于 · 免责声明 · 联系我们
```

**设计**

- 背景：--bg-secondary
- 字号：13px
- 颜色：--text-tertiary
- 对齐：居中
- 内边距：32px 上下

**原则**

- 只保留必要链接
- 不要社交媒体图标（现在没有）
- 不要多列布局（太复杂）

## 📱 响应式策略

### 移动端（< 640px）

**关键调整**

1. **Hero 区**

   - 标题：56px → 32px
   - 按钮：全宽（去掉左右 padding）
   - 内边距：160px → 80px

2. **快速说明**

   - 三列 → 单列
   - 每项间距：16px

3. **信任指标**

   - 数字：横向 → 纵向（如果空间不够）
   - 承诺图标：4 个 → 2×2 网格

4. **导航栏**
   - 隐藏链接，显示汉堡菜单
   - Logo 字号缩小

### 平板（641px - 1024px）

- 基本保持桌面布局
- 容器宽度：720px
- 字号略微缩小（-2px）

## ⏱️ 性能要求

### 首屏加载目标（关键指标）

- **FCP**（首次内容绘制）：< 1.0s
- **LCP**（最大内容绘制）：< 1.5s
- **TTI**（可交互时间）：< 2.0s

### 实现策略

1. **关键 CSS 内联**：Hero 区样式
2. **字体预加载**：Inter（仅 Regular + Bold）
3. **无图片**：首屏只用 Emoji 和文字
4. **代码分割**：地图库在点击后才加载

## 🎯 成功指标（数据驱动）

### 核心指标（必须追踪）

| 指标              | 目标值 | 说明                    |
| ----------------- | ------ | ----------------------- |
| **主 CTA 点击率** | > 50%  | Hero 区按钮点击率       |
| **跳出率**        | < 30%  | 首页跳出率              |
| **首屏停留时间**  | > 5 秒 | 说明用户在认真看        |
| **地图页转化率**  | > 80%  | 点击 CTA 后真的进入地图 |

### 次要指标

- 平均滚动深度：> 60%
- 移动端 CTA 点击率：> 40%
- 回访率（7 天）：> 15%

## 📊 与原方案对比

### 区块数量

| 对比项   | 原方案   | 精简方案    | 变化 |
| -------- | -------- | ----------- | ---- |
| 区块数   | 9 个     | 4 个        | -56% |
| 首屏高度 | ~1200px  | ~800px      | -33% |
| CTA 数量 | 3+       | 1 主 + 1 次 | -50% |
| 文字量   | ~500 字  | ~100 字     | -80% |
| 用户决策 | 多个选择 | 单一路径    | 清晰 |

### 用户路径

**原方案**：

```
看标题 → 读副标题 → 看数字 → 读价值 → 学习如何用 →
看动态 → 读承诺 → 最后才点 CTA
```

时间：30-60 秒，7 次滚动

**精简方案**：

```
看标题 → 点 CTA
```

时间：3-5 秒，0 次滚动

## 💡 产品思维总结

### 设计哲学

> **用户不是来"浏览"网站的，而是来"完成任务"的。**
>
> 首页的任务就是：**让用户尽快进入地图。**
>
> 一切让用户"停下来阅读"的内容，都是在阻碍这个目标。

### 删减的勇气

**我们删除了**：

- 长篇副标题
- 详细功能介绍
- 使用教程
- 最新动态
- 详细承诺说明
- 多个 CTA
- 装饰性元素

**我们保留了**：

- 一个清晰的标题
- 一个显眼的按钮
- 最少的信任背书
- 极简的功能说明

### 衡量成功的方式

**不是**：

- ❌ 页面有多美观
- ❌ 内容有多丰富
- ❌ 停留时间有多长

**而是**：

- ✅ 有多少人点击了 CTA
- ✅ 有多快点击了 CTA
- ✅ 点击后是否继续使用

## 📞 设计原则卡（随时参考）

在任何设计决策时，问自己：

1. ❓ 这个元素能帮助用户更快进入地图吗？

   - 如果不能 → 删除

2. ❓ 这段文字用户会认真读吗？

   - 如果不会 → 删除或缩短

3. ❓ 这个装饰能建立信任或传递情感吗？

   - 如果不能 → 删除

4. ❓ 这个按钮会分散主 CTA 的注意力吗？
   - 如果会 → 删除或降低优先级
